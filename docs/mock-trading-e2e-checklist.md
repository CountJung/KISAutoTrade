# 모의투자 E2E 체크리스트

> 실계좌 테스트 전 모의투자 프로필에서 국내/해외 자동매매 핵심 흐름을 분리 확인한다.

## 사전 조건

- Settings에서 모의투자 프로필을 활성화한다.
- 자동매매 대상 종목은 국내 1개, 해외 1개부터 시작한다.
- Settings → 리스크 관리에서 일일 주문 제한, 연속 손실 차단, 변동성 기반 수량 산정 상태를 확인한다.
- Log 페이지를 열고 `WARN`/`ERROR`가 반복되는지 관찰한다.

## 국내 주문

- Trading에서 국내 종목 현재가 조회가 성공한다.
- 국내 수동 매수 주문이 접수되고 주문번호가 기록된다.
- Dashboard 미체결 주문에 `대기/부분체결/잔여` 상태가 표시된다.
- 체결 조회 후 포지션 수량과 평균단가가 갱신된다.
- 국내 수동 매도 후 당일 체결 내역과 History 기록이 갱신된다.

## 해외 주문

- Trading에서 해외 종목 현재가와 거래소 코드가 조회된다.
- 해외 모의투자 주문은 지정가로만 제출한다.
- AMEX 또는 미지원 ETF 제한 메시지가 UI/Log에 명확히 표시된다.
- 해외 체결 기록에는 USD 값과 KRW 환산값, 체결 시점 환율이 함께 저장된다.
- 해외 포지션은 국내 PositionTracker와 섞이지 않는다.

## 자동매매

- Strategy에서 테스트 전략 1개만 켜고 대상 종목을 제한한다.
- 자동매매 시작 시 국내/해외 잔고 기반으로 전략 포지션 상태가 복원된다.
- 자동매매 시작 시 ATR 준비 종목 수가 Settings 리스크 관리에 표시된다.
- 매수 신호는 TradeGuard, 일일 주문 제한, 연속 손실 차단, ATR 수량 산정을 통과한 뒤에만 주문된다.
- 주문번호 기반 체결 조회가 우선 실행되고, 반영 지연 시 다음 틱 가격 fallback이 로그에 남는다.

## 종료 기준

- `cargo check`, `cargo test`, `npx tsc --noEmit`, `npm run check:fsd`가 통과한다.
- Dashboard/History/Log에서 주문 접수, 체결, 포지션, 통계가 서로 일치한다.
- 같은 종목의 반복 매수/매도 루프가 발생하지 않는다.
