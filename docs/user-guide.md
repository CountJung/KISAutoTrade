## 1. 프로젝트 개요

KISAutoTrade는 **한국투자증권(KIS) Open API**를 이용해 주식을 자동으로 매매하는 개인용 데스크탑 앱입니다.

### 핵심 특징

| 기능 | 설명 |
|------|------|
| **멀티 계좌 프로필** | 실전·모의투자 앱 키를 별도 프로필로 등록 · 전환 |
| **11가지 자동매매 전략** | 이동평균·RSI·모멘텀·이격도·52주신고가·연속상승·돌파실패·강한종가·변동성확장·평균회귀·추세필터 |
| **리스크 관리** | 일일 손실 한도 초과 시 비상 정지 자동 발동, 종목당 최대 비중 제한 |
| **실시간 시세** | KIS WebSocket (`H0STCNT0`) 체결 스트림으로 틱 단위 전략 판단 |
| **Discord 알림** | 체결·에러·일별 요약 알림 (Webhook 없이 Bot 토큰 직접 연동) |
| **JSON 파일 저장** | DB 없이 연/월/일 폴더 구조로 거래 기록·통계 자동 저장 |
| **웹 접속 모드** | 앱 실행 중 브라우저에서 `http://localhost:7474` 로 UI 접근 가능 |

### 지원 플랫폼

- **Windows** 10 / 11 (x64)
- **macOS** 12 Monterey 이상 (Apple Silicon + Intel Universal)

```
│  └──────┬──────┘  └──────┬───────┘  └───────┬───────┘  │
│         │                │                  │           │

│  ┌─────────────────────────────────────────────────┐   │
```

### 자동매매 실행 흐름

자동매매 시작 버튼을 누르면 다음 순서로 동작합니다:

```
1. 장 시간 감지 (09:00~15:30 KST, 모의투자도 동일)
       │
2. KIS WebSocket 연결 → 전략에 등록된 모든 종목 구독 (H0STCNT0)
       │
3. 매 틱마다 전략 판단 (on_tick)
       │
4. 매매 신호 발생 → RiskManager 검증
   ├─ 비상 정지 상태? → 차단
   ├─ 손실 한도 초과? → 차단
   └─ 종목 비중 한도 초과? → 차단
       │
5. KIS REST API place_order() 호출
       │
6. 미체결 주문 등록 → 체결 이벤트 수신 후 PositionTracker 업데이트
       │
7. JSON 파일 기록 (거래·통계)
       │
8. Discord 체결 알림 전송
```

### 데이터 저장 위치

| OS | 앱 데이터 폴더 |
|----|---------------|
| Windows | `%APPDATA%\com.countjung.kisautotrade\` |
| macOS | `~/Library/Application Support/com.countjung.kisautotrade/` |

```
com.countjung.kisautotrade/
├── profiles.json              ← 계좌 프로필 (API 키 포함, 암호화 없음)
└── data/
    ├── trades/YYYY/MM/DD/     ← 체결 내역
    ├── orders/YYYY/MM/DD/     ← 주문 기록
    ├── stats/YYYY/MM/DD/      ← 일별 통계
    ├── balance/YYYY/MM/DD/    ← 잔고 스냅샷
    └── strategies/{profile_id}/strategies.json  ← 전략 설정 (프로필별 독립)
```

> ⚠️ `profiles.json`에 APP SECRET이 **평문**으로 저장됩니다. 파일을 타인과 공유하지 마세요.

## 3. 화면별 사용 설명

### Dashboard

- 실시간 **잔고 카드**: 총 평가금액, 총 손익, 보유 현금 표시
- **포지션 테이블**: 보유 종목별 매입가·현재가·평가손익
- **당일 체결 내역**: 오늘 체결된 매수/매도 목록
- **WebSocket 연결 상태 Chip**: 초록(연결됨) / 회색(끊김)
- **미체결 주문**: 대기 중인 주문 목록

### Trading

- **종목 현재가 조회**: 종목코드 입력 후 돋보기 버튼 → 현재가·전일비 조회
- **수동 매수/매도**: 지정가(가격 직접 입력) 또는 시장가 주문
- 주문 후 체결 내역 자동 갱신 (약 1.5초 지연)

### Strategy

- 전략 ON/OFF 스위치 및 파라미터 설정 (상세 내용은 [5장](#5-전략-페이지-자동매매-세팅) 참고)

### History

- 날짜 범위 선택 → 해당 기간 거래 기록 조회
- 일별 통계 테이블: 매수금액·매도금액·실현손익·거래 건수

### Log

- 로그 레벨 필터 (`DEBUG` / `INFO` / `WARN` / `ERROR`)
- 키워드 검색
- 높이 드래그 리사이즈 가능

### Settings

- **계좌 프로필 관리**: KIS 앱 키 · 시크릿 · 계좌번호 · 실전/모의 설정
- **테마**: 라이트 / 다크 / 시스템 연동
- **웹 포트**: 앱 내 axum 웹 서버 포트 설정 (기본 7474)
- **Discord 연동**: 봇 토큰·채널 ID 설정 및 테스트 메시지 발송
- **API 디버그 로그**: ON 시 KIS API 요청/응답 전문을 로그에 기록

## 4. GitHub Actions 자동 빌드 / 릴리즈

프로젝트는 `.github/workflows/release.yml`에 자동 빌드·릴리즈 워크플로가 설정되어 있습니다.  
**`v`로 시작하는 Git 태그를 push하면** Windows (.msi / NSIS .exe)와 macOS Universal (.dmg)를 자동 빌드하여 GitHub Releases에 드래프트로 등록합니다.

### 4-1. GitHub Repository Secrets 설정

GitHub Actions가 릴리즈를 생성하려면 별도의 Secrets 설정이 **필요 없습니다.**  
내장 `GITHUB_TOKEN`이 자동으로 사용됩니다.

> macOS 코드 사이닝이 필요한 경우 `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD` 등의 Secrets 추가가 필요하지만 **현재 워크플로는 서명 없이 동작**합니다.  
> Apple Gatekeeper로 인해 macOS에서 최초 실행 시 "확인할 수 없음" 경고가 나올 수 있습니다 — 시스템 환경설정 → 보안 → 확인 없이 열기로 해결합니다.

### 4-2. 릴리즈 절차

#### ① 버전 번호 결정

버전은 `src-tauri/Cargo.toml`과 `src-tauri/tauri.conf.json` 두 파일에 정의되어 있습니다.  
두 파일의 버전을 일치시킨 후 커밋합니다.

```toml
# src-tauri/Cargo.toml
[package]
version = "0.2.0"   # ← 여기 변경
```

```json
// src-tauri/tauri.conf.json
{
  "version": "0.2.0"   // ← 여기도 동일하게 변경
}
```

```bash
git add src-tauri/Cargo.toml src-tauri/tauri.conf.json
git commit -m "chore: bump version to 0.2.0"
git push origin main
```

#### ② Git 태그 생성 및 push

```bash
# 태그 생성 (v 접두사 + Semantic Versioning)
git tag v0.2.0

# 태그 push → GitHub Actions 자동 트리거
git push origin v0.2.0
```

> 방금 실행한 `git push origin v0.1.0`처럼 단일 명령으로 push할 수 있습니다.

#### ③ GitHub Actions 진행 확인

1. GitHub 저장소 → **Actions** 탭 클릭
2. `Release` 워크플로 실행 중 확인 (노란 원 = 빌드 중)
3. Windows 빌드(약 15~20분)와 macOS 빌드(약 20~30분)가 병렬 실행됨
4. 두 플랫폼 모두 완료되면 초록 체크 표시

빌드 실패 시 해당 Job 클릭 → 오류 로그 확인 가능합니다.

#### ④ 릴리즈 게시

1. GitHub 저장소 → **Releases** 탭
2. **Draft** 상태의 릴리즈 클릭
3. 릴리즈 노트 내용 수정 (주요 변경사항 기재)
4. **Publish release** 클릭 → 공개 릴리즈로 전환

#### ⑤ 릴리즈 결과물 (Artifacts)

| 파일 | 플랫폼 | 설명 |
|------|--------|------|
| `KISAutoTrade_0.2.0_x64-setup.exe` | Windows | NSIS 설치 파일 (권장) |
| `KISAutoTrade_0.2.0_x64_en-US.msi` | Windows | MSI 설치 파일 |
| `KISAutoTrade_0.2.0_universal.dmg` | macOS | Universal (Apple Silicon + Intel) |

### 4-3. 태그 삭제 (잘못 push한 경우)

```bash
# 로컬 태그 삭제
git tag -d v0.2.0

# 원격 태그 삭제
git push origin --delete v0.2.0
```

삭제 후 재생성하면 Actions가 다시 트리거됩니다.

### 4-4. 워크플로 구조 요약

```yaml
# .github/workflows/release.yml 핵심 구조
on:
  push:
    tags: ['v[0-9]+.[0-9]+.[0-9]+']   # v숫자.숫자.숫자 태그에만 반응

jobs:
  build-tauri:
    matrix:
      - platform: windows-latest        # Windows .msi / .exe
      - platform: macos-latest          # macOS Universal .dmg
    steps:
      - Checkout
      - Setup Node.js 20
      - Setup Rust (stable)
      - Cache Rust build (Swatinem/rust-cache)
      - npm install
      - tauri-apps/tauri-action@v0      # 빌드 + GitHub Release 업로드
```

## 5. 전략 페이지 자동매매 세팅

### 5-1. 전체 흐름 요약

```
① Settings에서 계좌 프로필 설정 (모의투자 우선 권장)
       ↓
② Strategy 페이지 → 종목 선택 패널에서 종목 검색
       ↓
③ 원하는 전략 카드에 종목 추가
       ↓
④ 전략 파라미터(기간·수량 등) 설정 후 저장
       ↓
⑤ 전략 ON/OFF 스위치로 활성화
       ↓
⑥ Dashboard 또는 Trading 페이지에서 자동매매 시작 버튼 클릭
```

---

### 5-2. 사전 준비: 계좌 프로필 설정

**Settings** 페이지 → **계좌 프로필** 섹션:

| 항목 | 설명 | 예시 |
|------|------|------|
| 프로필 이름 | 식별용 이름 | `모의투자`, `실전-삼성` |
| APP KEY | KIS Developers에서 발급 | `PSxxxxxxxxxxxxxxxx` |
| APP SECRET | KIS Developers에서 발급 | (60자 내외 문자열) |
| 계좌번호 | 하이픈 있어도 없어도 무방 | `50123456-01` 또는 `5012345601` |
| 모의투자 여부 | 토글 스위치 | ✅ 처음에는 반드시 ON |

> **⚠️ 처음 사용 시 반드시 모의투자 프로필로 시작하세요!** 실전 계좌 실수 주문 방지.

### 5-3. 종목 선택 패널 사용법

Strategy 페이지 상단에 **종목 선택** 패널이 있습니다.

#### 국내 주식

1. **"국내"** 토글 버튼 선택 (기본값)
2. 6자리 종목코드 입력 (예: `005930`)
   - 숫자 6자리: 일반 주식
   - 영숫자 혼합 6자리: ETF (예: `069500`, `0005A0`)
3. 드롭다운에서 종목 클릭 → 선택됨 표시

#### 해외(미국) 주식

1. **"해외 (미국)"** 토글 버튼 선택
2. 영문 티커 입력 (예: `AAPL`, `TSLA`, `NVDA`)
3. **검색** 버튼 클릭 → NAS(나스닥) → NYS(뉴욕) → AMEX 순서로 자동 탐색
4. 종목명이 자동으로 표시됨

### 5-5. 11가지 전략 상세 설명 및 세팅 예시

#### 전략 1: 이동평균 교차 (`ma_cross`)

**설명**: 단기 이동평균이 장기 이동평균을 상향 돌파(골든크로스) 시 매수, 하향 돌파(데스크로스) 시 매도.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 단기 MA | 5 | 2 ~ 50 | 단기 이동평균 기간 (봉 수) |
| 장기 MA | 20 | 5 ~ 200 | 장기 이동평균 기간 (봉 수) |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

**세팅 예시** — 삼성전자 보수적 장기 설정:
```
종목: 005930 (삼성전자)
단기 MA: 10
장기 MA:  50
주문수량: 5
→ 저장 → ON
```

---

#### 전략 2: RSI (`rsi`)

**설명**: RSI가 과매도 기준 이하에서 반등 시 매수, 과매수 기준 이상에서 하락 시 매도.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| RSI 기간 | 14 | 5 ~ 50 | RSI 계산 기간 |
| 과매도 기준 | 30 | 10 ~ 40 | 이 이하 → 반등 시 매수 신호 |
| 과매수 기준 | 70 | 60 ~ 90 | 이 이상 → 하락 시 매도 신호 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

**세팅 예시** — KODEX 200 ETF 단기 역추세:
```
종목: 069500 (KODEX 200)
RSI 기간: 14
과매도 기준: 25
과매수 기준: 75
주문수량: 3
→ 저장 → ON
```

---

#### 전략 3: 모멘텀 (`momentum`)

**설명**: N기간 전 가격 대비 현재가 변화율이 임계값 이상이면 매수, 이하이면 매도 (추세 추종).

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 비교 기간 | 20 | 5 ~ 60 | 과거 몇 봉 전과 비교할지 |
| 임계값 (%) | 5.0 | 1 ~ 20 | 매매 발동 최소 변화율 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

**세팅 예시** — 테슬라 강한 추세 추종:

#### 전략 4: 이격도 (`deviation`)

**설명**: 현재가가 이동평균 대비 일정 % 이하이면 매수(저평가), 이상이면 매도(고평가).

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| MA 기간 | 20 | 5 ~ 120 | 기준 이동평균 기간 |
| 매수 이격 (%) | -5.0 | -20 ~ -1 | MA 대비 이 % 이하이면 매수 (음수) |
| 매도 이격 (%) | 5.0 | 1 ~ 20 | MA 대비 이 % 이상이면 매도 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

**세팅 예시** — POSCO홀딩스 평균 회귀 기대:
```

매도 이격:  5.0%  (MA 대비 5% 이상 상승 시 매도)
주문수량: 2
→ 저장 → ON
```

#### 전략 5: 52주 신고가 (`fifty_two_week_high`)

**설명**: 최근 252 거래일(약 1년) 최고가를 재돌파하면 매수. 매수 후 지정 % 하락 시 손절.  
자동매매 시작 시 KIS 차트 API로 과거 데이터를 자동 초기화합니다.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 조회 기간 (거래일) | 252 | 60 ~ 504 | 52주 신고가 계산 거래일 수 |
| 손절 기준 (%) | 3.0 | 1 ~ 15 | 매수가 대비 이 % 이상 하락 시 손절 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

#### 전략 6: 연속 상승/하락 (`consecutive_move`)

**설명**: N일 연속 종가 상승 시 매수, M일 연속 하락 시 매도.

#### 전략 7: 돌파 실패 (`failed_breakout`)

**설명**: 최근 N일 전고점을 버퍼% 이상 돌파하면 매수. 이후 전고점 이하로 내려오면 돌파 실패로 즉시 매도.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 전고점 기간 | 20 | 5 ~ 60 | 전고점 계산 기간 (거래일) |
| 돌파 버퍼 (%) | 0.5 | 0.1 ~ 5.0 | 전고점 대비 추가 % 이상 시 돌파 인정 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |
