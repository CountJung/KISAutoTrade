## 1. 프로젝트 개요

KISAutoTrade는 **한국투자증권(KIS) Open API**를 이용해 주식을 자동으로 매매하는 개인용 데스크탑 앱입니다.

### 핵심 특징

| 기능 | 설명 |
|------|------|
| **멀티 계좌 프로필** | 실전·모의투자 앱 키를 별도 프로필로 등록 · 전환 |
| **11가지 자동매매 전략** | 이동평균·RSI·모멘텀·이격도·52주신고가·연속상승·돌파실패·강한종가·변동성확장·평균회귀·추세필터 |
| **리스크 관리** | 일일 손실 한도 초과 시 비상 정지 자동 발동, 종목당 최대 비중 제한 |
| **실시간 시세** | KIS WebSocket (`H0STCNT0`) 체결 스트림으로 틱 단위 전략 판단 |
| **Discord 알림** | 체결·에러·일별 요약 알림 (Webhook 없이 Bot 토큰 직접 연동) |
| **JSON 파일 저장** | DB 없이 연/월/일 폴더 구조로 거래 기록·통계 자동 저장 |
| **웹 접속 모드** | 앱 실행 중 브라우저에서 `http://localhost:7474` 로 UI 접근 가능 |

### 지원 플랫폼

- **Windows** 10 / 11 (x64)
- **macOS** 12 Monterey 이상 (Apple Silicon + Intel Universal)

### 자동매매 실행 흐름

자동매매 시작 버튼을 누르면 다음 순서로 동작합니다:

```
1. 장 시간 감지 (09:00~15:30 KST, 모의투자도 동일)
       │
2. KIS WebSocket 연결 → 전략에 등록된 모든 종목 구독 (H0STCNT0)
       │
3. 매 틱마다 전략 판단 (on_tick)
       │
4. 매매 신호 발생 → RiskManager 검증
   ├─ 비상 정지 상태? → 차단
   ├─ 손실 한도 초과? → 차단
   └─ 종목 비중 한도 초과? → 차단
       │
5. KIS REST API place_order() 호출
       │
6. 미체결 주문 등록 → 체결 이벤트 수신 후 PositionTracker 업데이트
       │
7. JSON 파일 기록 (거래·통계)
       │
8. Discord 체결 알림 전송
```

### 데이터 저장 위치

| OS | 앱 데이터 폴더 |
|----|---------------|
| Windows | `%APPDATA%\com.countjung.kisautotrade\` |
| macOS | `~/Library/Application Support/com.countjung.kisautotrade/` |

```
com.countjung.kisautotrade/
├── profiles.json              ← 계좌 프로필 (API 키 포함, 암호화 없음)
└── data/
    ├── trades/YYYY/MM/DD/     ← 체결 내역
    ├── orders/YYYY/MM/DD/     ← 주문 기록
    ├── stats/YYYY/MM/DD/      ← 일별 통계
    ├── balance/YYYY/MM/DD/    ← 잔고 스냅샷
    └── strategies/{profile_id}/strategies.json  ← 전략 설정 (프로필별 독립)
```

> ⚠️ `profiles.json`에 APP SECRET이 **평문**으로 저장됩니다. 파일을 타인과 공유하지 마세요.

## 3. 화면별 사용 설명

### Dashboard

- 실시간 **잔고 카드**: 총 평가금액, 총 손익, 보유 현금 표시
- **포지션 테이블**: 보유 종목별 매입가·현재가·평가손익
- **당일 체결 내역**: 오늘 체결된 매수/매도 목록
- **WebSocket 연결 상태 Chip**: 초록(연결됨) / 회색(끊김)
- **미체결 주문**: 대기 중인 주문 목록

### Trading

- **종목 현재가 조회**: 종목코드 입력 후 돋보기 버튼 → 현재가·전일비 조회
- **수동 매수/매도**: 지정가(가격 직접 입력) 또는 시장가 주문
- 주문 후 체결 내역 자동 갱신 (약 1.5초 지연)

### Strategy

- 전략 ON/OFF 스위치 및 파라미터 설정 (상세 내용은 [5장](#5-전략-페이지-자동매매-세팅) 참고)

### History

- 날짜 범위 선택 → 해당 기간 거래 기록 조회
- 일별 통계 테이블: 매수금액·매도금액·실현손익·거래 건수

### Log

- 로그 레벨 필터 (`DEBUG` / `INFO` / `WARN` / `ERROR`)
- 키워드 검색
- 높이 드래그 리사이즈 가능

### Settings

- **계좌 프로필 관리**: KIS 앱 키 · 시크릿 · 계좌번호 · 실전/모의 설정
- **테마**: 라이트 / 다크 / 시스템 연동
- **웹 포트**: 앱 내 axum 웹 서버 포트 설정 (기본 7474)
- **Discord 연동**: 봇 토큰·채널 ID 설정 및 테스트 메시지 발송
- **API 디버그 로그**: ON 시 KIS API 요청/응답 전문을 로그에 기록

## 4. GitHub Actions 자동 빌드 / 릴리즈

프로젝트는 `.github/workflows/release.yml`에 자동 빌드·릴리즈 워크플로가 설정되어 있습니다.  
**`v`로 시작하는 Git 태그를 push하면** Windows (.msi / NSIS .exe)와 macOS Universal (.dmg)를 자동 빌드하여 GitHub Releases에 드래프트로 등록합니다.

### 4-1. GitHub Repository Secrets 설정

GitHub Actions가 릴리즈를 생성하려면 별도의 Secrets 설정이 **필요 없습니다.**  
내장 `GITHUB_TOKEN`이 자동으로 사용됩니다.

> macOS 코드 사이닝이 필요한 경우 `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD` 등의 Secrets 추가가 필요하지만 **현재 워크플로는 서명 없이 동작**합니다.  
> Apple Gatekeeper로 인해 macOS에서 최초 실행 시 "확인할 수 없음" 경고가 나올 수 있습니다 — 시스템 환경설정 → 보안 → 확인 없이 열기로 해결합니다.

### 4-2. 릴리즈 절차

#### ① 버전 번호 결정

버전은 `src-tauri/Cargo.toml`과 `src-tauri/tauri.conf.json` 두 파일에 정의되어 있습니다.  
두 파일의 버전을 일치시킨 후 커밋합니다.

```toml
# src-tauri/Cargo.toml
[package]
version = "0.2.0"   # ← 여기 변경
```

```json
// src-tauri/tauri.conf.json
{
  "version": "0.2.0"   // ← 여기도 동일하게 변경
}
```

```bash
git add src-tauri/Cargo.toml src-tauri/tauri.conf.json
git commit -m "chore: bump version to 0.2.0"
git push origin main
```

#### ② Git 태그 생성 및 push

```bash
# 태그 생성 (v 접두사 + Semantic Versioning)
git tag v0.2.0

# 태그 push → GitHub Actions 자동 트리거
git push origin v0.2.0
# 통합 커맨드 - cargo.toml, tauri.conf.json 파일 버전 맞춤
git tag v0.1.2 && git push origin v0.1.2
```

> 방금 실행한 `git push origin v0.1.0`처럼 단일 명령으로 push할 수 있습니다.

#### ③ GitHub Actions 진행 확인

1. GitHub 저장소 → **Actions** 탭 클릭
2. `Release` 워크플로 실행 중 확인 (노란 원 = 빌드 중)
3. Windows 빌드(약 15~20분)와 macOS 빌드(약 20~30분)가 병렬 실행됨
4. 두 플랫폼 모두 완료되면 초록 체크 표시

빌드 실패 시 해당 Job 클릭 → 오류 로그 확인 가능합니다.

#### ④ 릴리즈 게시

1. GitHub 저장소 → **Releases** 탭
2. **Draft** 상태의 릴리즈 클릭
3. 릴리즈 노트 내용 수정 (주요 변경사항 기재)
4. **Publish release** 클릭 → 공개 릴리즈로 전환

#### ⑤ 릴리즈 결과물 (Artifacts)

| 파일 | 플랫폼 | 설명 |
|------|--------|------|
| `KISAutoTrade_0.2.0_x64-setup.exe` | Windows | NSIS 설치 파일 (권장) |
| `KISAutoTrade_0.2.0_x64_en-US.msi` | Windows | MSI 설치 파일 |
| `KISAutoTrade_0.2.0_universal.dmg` | macOS | Universal (Apple Silicon + Intel) |

### 4-3. 태그 삭제 (잘못 push한 경우)

```bash
# 로컬 태그 삭제
git tag -d v0.2.0

# 원격 태그 삭제
git push origin --delete v0.2.0
```

삭제 후 재생성하면 Actions가 다시 트리거됩니다.

### 4-4. 워크플로 구조 요약

```yaml
# .github/workflows/release.yml 핵심 구조
on:
  push:
    tags: ['v[0-9]+.[0-9]+.[0-9]+']   # v숫자.숫자.숫자 태그에만 반응

jobs:
  build-tauri:
    matrix:
      - platform: windows-latest        # Windows .msi / .exe
      - platform: macos-latest          # macOS Universal .dmg
    steps:
      - Checkout
      - Setup Node.js 20
      - Setup Rust (stable)
      - Cache Rust build (Swatinem/rust-cache)
      - npm install
      - tauri-apps/tauri-action@v0      # 빌드 + GitHub Release 업로드
```


### 5-2. 사전 준비: 계좌 프로필 설정

**Settings** 페이지 → **계좌 프로필** 섹션:

| 항목 | 설명 | 예시 |
|------|------|------|
| 프로필 이름 | 식별용 이름 | `모의투자`, `실전-삼성` |
| APP KEY | KIS Developers에서 발급 | `PSxxxxxxxxxxxxxxxx` |
| APP SECRET | KIS Developers에서 발급 | (60자 내외 문자열) |
| 계좌번호 | 하이픈 있어도 없어도 무방 | `50123456-01` 또는 `5012345601` |
| 모의투자 여부 | 토글 스위치 | ✅ 처음에는 반드시 ON |

> **⚠️ 처음 사용 시 반드시 모의투자 프로필로 시작하세요!** 실전 계좌 실수 주문 방지.

### 5-3. 종목 선택 패널 사용법

Strategy 페이지 상단에 **종목 선택** 패널이 있습니다.

#### 국내 주식

1. **"국내"** 토글 버튼 선택 (기본값)
2. 6자리 종목코드 입력 (예: `005930`)
   - 숫자 6자리: 일반 주식
   - 영숫자 혼합 6자리: ETF (예: `069500`, `0005A0`)
3. 드롭다운에서 종목 클릭 → 선택됨 표시

#### 해외(미국) 주식

1. **"해외 (미국)"** 토글 버튼 선택
2. 영문 티커 입력 (예: `AAPL`, `TSLA`, `NVDA`)
3. **검색** 버튼 클릭 → NAS(나스닥) → NYS(뉴욕) → AMEX 순서로 자동 탐색
4. 종목명이 자동으로 표시됨

### 5-4. 전략 카드 구성

각 전략은 독립된 카드로 표시됩니다:

```
┌─────────────────────────────────────────────────────────┐
│ [전략 이름]                      [ON/OFF 스위치]        │
│ 전략 설명 한 줄                                         │
├─────────────────────────────────────────────────────────┤
│ 📌 대상 종목                                            │
│  [KODEX200 069500] [X]   [ACE200 360750] [X]           │
│  [추가] 버튼 (종목 선택 패널에서 선택 후 클릭)          │
├─────────────────────────────────────────────────────────┤
│ 📋 파라미터                                             │
│  단기 MA [5 ▲▼]  장기 MA [20 ▲▼]  주문수량 [1 ▲▼]     │
├─────────────────────────────────────────────────────────┤
│                                             [저장]       │
└─────────────────────────────────────────────────────────┘
```

> 💡 **하나의 전략 카드에 여러 종목을 등록하면 각 종목의 가격 이력과 포지션 상태가 독립적으로 관리됩니다.**  
> 동일 성격의 ETF(예: KODEX 200 + ACE 200)도 같은 카드에 등록해 동시에 운용할 수 있습니다.  
> 단, [가격 조건 매매(전략 12)](#전략-12-가격-조건-매매-price_condition) 외의 전략은 **주문 수량이 전략 카드 전체에서 공유**됩니다.

### 5-5. 11가지 전략 상세 설명 및 세팅 예시

#### 전략 1: 이동평균 교차 (`ma_cross`)

**설명**: 단기 이동평균이 장기 이동평균을 상향 돌파(골든크로스) 시 매수, 하향 돌파(데스크로스) 시 매도.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| 단기 MA | 5 | 2 ~ 50 | 단기 이동평균 기간 (봉 수) |
| 장기 MA | 20 | 5 ~ 200 | 장기 이동평균 기간 (봉 수) |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

**세팅 예시** — 삼성전자 보수적 장기 설정:
```
종목: 005930 (삼성전자)
단기 MA: 10
장기 MA:  50
주문수량: 5
→ 저장 → ON
```

---

#### 전략 2: RSI (`rsi`)

**설명**: RSI가 과매도 기준 이하에서 반등 시 매수, 과매수 기준 이상에서 하락 시 매도.

| 파라미터 | 기본값 | 범위 | 설명 |
|----------|--------|------|------|
| RSI 기간 | 14 | 5 ~ 50 | RSI 계산 기간 |
| 과매도 기준 | 30 | 10 ~ 40 | 이 이하 → 반등 시 매수 신호 |
| 과매수 기준 | 70 | 60 ~ 90 | 이 이상 → 하락 시 매도 신호 |
| 주문수량 | 1 | 1 ~ | 1회 매수/매도 수량 (주) |

| 모의투자 | 최대 2개 (국내만) | 초당 2건 제한 엄격 |
| 실전투자 | 최대 4~5개 | 초당 20건으로 여유 있음 |

- **해외 종목은 국내 장 마감 후 단독 운용** (별도 프로필 추천)
- 종목을 줄이면 레이트 리밋 오류가 극적으로 감소합니다

#### ✅ 폴링 주기 조정 (현재 10초 기본)

종목이 2개 이상이면 10초 폴링은 모의투자에서 레이트 리밋 초과를 유발합니다.

```
모의투자 + 종목 3개 이상 → 폴링 30초 권장
실전투자 + 종목 5개 이하 → 폴링 10~15초 유지
```

### 6-3. 상승장에 적합한 전략 조합

#### 🟢 상승장 강력 추천 전략 TOP 4

| 순위 | 전략 | 이유 | 핵심 파라미터 조정 |
|------|------|------|--------------|
| 1 | **이격도** (`deviation`) | 일시적 눌림목에서 저점 매수 → 추세 재개 시 수익 | `buy_threshold: -5 ~ -8%`, MA 기간 20 |
| 2 | **변동성 확장** (`volatility_expansion`) | 급등 초반부 포착 → 추세 지속 구간에서 탑승 | `expansion_factor: 1.5` (더 민감), 손절 3% |
| 3 | **강한 종가** (`strong_close`) | 전일 강하게 마감 → 당일 갭업 추종 | `close_strength: 2.0%`, 손절 3% |
| 4 | **추세 필터** (`trend_filter`) | 장기 MA 위에서만 매수 → 하락장 노출 없음 | 장기 200, 단기 5, 중기 20 |

#### 🟡 보조적으로 활용 가능

| 전략 | 상승장 적합성 | 주의사항 |
|------|-------------|---------|
| **모멘텀** (`momentum`) | ✅ 추세 추종 | 임계값 낮추면(3%) 민감도 증가 |
| **52주 신고가** (`fifty_two_week_high`) | ✅ 돌파 매수 | 한 번 매수 후 손절선 중요 |

#### 🔴 상승장 비권장 (비활성화 추천)

| 전략 | 이유 |
|------|------|
| **MA교차** (`ma_cross`) | 기본값(단기 5, 장기 20) 기준 신호 지연 — 상승이 끝날 즈음 매수 신호 발생 가능 |
| **RSI** (`rsi`) | 상승장에서 과매수 상태 유지 → 매도 신호 과다 |
| **연속상승/하락** (`consecutive_move`) | 노이즈 많음, 상승 중 눌림목에서 오시그널 |
| **돌파실패** (`failed_breakout`) | 역추세 전략 — 상승장에서 역방향 베팅 위험 |
| **평균회귀** (`mean_reversion`) | 상승 추세 도중 단기 과매도를 역추세로 오판 가능 |

### 6-3-1. 상승장에서 MA교차·추세필터 이득을 못 보는 원인과 파라미터 개선

> **상황**: 9시 30분부터 명확한 상승 추세였는데 MA교차 + 추세필터로 이득을 거의 못 봤다면 아래 원인을 체크하세요.

#### 원인 ①: MA 기간이 너무 길어 신호가 늦게 발생

MA교차 기본값(단기 5봉, 장기 20봉)은 **일봉** 기준으로 설계되었습니다.
현재 시스템은 **실시간 틱(10초 폴링)**에 반응하므로, 5봉·20봉이 쌓이는 데 각각 50초·200초가 필요합니다.  
장중 상승 추세가 이미 한참 진행된 뒤에야 골든크로스가 발생해 매수 타이밍을 놓칩니다.

**개선 방법 — 단기 틱 전략용 파라미터:**

```
[MA교차 — 장중 상승 추세 추종용]
단기 MA: 3  (30초 분량)
장기 MA:  8  (80초 분량)
주문수량: 1
```

> 단기·장기 차이가 좁을수록 민감도는 높아지지만 노이즈 신호도 증가합니다.  
> 변동성이 큰 종목은 단기 3·장기 10 조합을 시도해보세요.

#### 원인 ②: 추세 필터의 장기 MA(200봉)가 채워지지 않음

추세 필터는 자동매매 **시작 시** 과거 종가로 `long_period`만큼의 버퍼를 적재합니다.  
단, **틱이 쌓이는 것은 실시간**이므로 장기 200봉은 `200 × 10초 = 2000초(약 33분)` 이후부터 유효합니다.  
9시 30분에 시작했다면 10시 3분경에야 장기 MA가 유의미해집니다.

**확인 방법:**
- Log 페이지 → "추세 필터 초기화 [종목코드]: 과거 N개 가격 로드" 메시지 확인
- N이 200 미만이면 아직 신호가 발생하지 않음 (Hold만 반환)

**개선 방법 — 틱 기반 단기 추세 필터:**

```
[추세 필터 — 장중 빠른 신호용]
장기 MA: 40   (약 6~7분)
중기 MA: 10   (약 1.5분)
단기 MA:  3   (30초)
주문수량: 1
```

> 장기 40봉이면 시작 후 약 7분 내에 필터가 활성화됩니다.  
> 단, 장기 MA가 짧을수록 하락 기간 중 필터 효과가 약해지므로 손절 기준을 함께 설정하세요.

#### 원인 ③: 두 전략의 신호 조건이 겹쳐 상호 충돌

MA교차와 추세 필터를 동시에 ON하면 **같은 종목에 두 전략이 독립적으로 주문**을 냅니다.  
추세 필터가 매수했는데 MA교차가 데드크로스 감지 후 매도를 내면 포지션이 조기 청산될 수 있습니다.

**권장 운용 방식:**

| 상황 | 권장 전략 조합 |
|------|--------------|
| 장중 빠른 추세 추종 | MA교차만 사용 (단기 3, 장기 8) |
| 일별 추세 + 장기 필터 | 추세 필터만 사용 (장기 40~100, 중기 10, 단기 3) |
| 두 전략 함께 쓸 때 | 서로 다른 종목에 각각 할당 |

#### 상승장 MA교차 최적화 세팅 예시

```
[상황: 9시 30분 상승 추세 — 빠른 골든크로스 포착]

종목: 069500 (KODEX 200) 또는 360750 (TIGER 미국 S&P500)
단기 MA: 3
장기 MA:  8
주문수량: 2
→ 저장 → ON

[추세 필터 — 동일 종목에 함께 사용 시]
장기 MA: 30   ← 약 5분 내 활성화
중기 MA:  8
단기 MA:  3
주문수량: 1
→ 저장 → ON
```

> ⚠️ 틱 기반으로 단기화할수록 **잦은 매수·매도가 발생해 수수료 비용이 누적**됩니다.  
> History 탭 → 수수료 항목을 주기적으로 확인하고, 신호 빈도 대비 실수익을 점검하세요.

> 별도로 전략을 OFF 할 필요가 없습니다. (→ [섹션 7. 시장 시간표 자동 제어](#7-시장-시간표-자동-제어) 참고)

## 7. 시장 시간표 자동 제어

자동매매 폴링 루프는 각 종목이 속한 시장의 개장 여부를 **실시간으로 확인**하여
폐장 중에는 KIS API 호출 자체를 건너뜁니다.
이를 통해 불필요한 API 콜 낭비와 EGW00201 rate-limit 오류를 예방합니다.

### 7-1. 지원 시장 및 개장 시간

| 시장 | 종목 조건 | 정규 개장 시간 (KST) |
|------|----------|---------------------|
| **KRX (국내)** | 6자리 숫자로 시작 (예: `069500`, `360750`) | 월~금 09:00 ~ 15:30 |
| **US (해외)** | 영문 포함 또는 KRX 외 코드 (예: `QQQM`, `SPYM`) | 월~금 22:00 ~ 익일 07:00 (EDT 기준 버퍼 포함) |

> **KST = UTC+9** 기준. 서머타임(EDT/EST) 전환 시에도 22:00-07:00 창이 양쪽 시즌을 커버합니다.
> - EDT(UTC-4) 정규장: 23:30 ~ 06:00 KST
> - EST(UTC-5) 정규장: 00:30 ~ 07:00 KST

### 7-2. 동작 방식

```
자동매매 10초 폴링 루프
  │
  ├─ [사전 체크] 모든 활성 종목의 시장이 폐장 중?
  │     YES → 로그: "모든 시장 폐장 (KRX·US 모두 폐장) — 5분 대기 후 재확인"
  │             5분 sleep 후 재진입 (API 호출 0건)
  │     NO  → 개장 종목만 순회
  │
  └─ [종목별 체크] 해당 종목 시장이 폐장 중?
        YES → debug 로그: "시장 폐장 — 건너뜀: QQQM (US)"
              다음 종목으로 continue (API 호출 0건)
        NO  → 현재가 조회 + 전략 신호 처리 (정상 동작)
```

### 7-3. 로그 확인 방법

Log 페이지에서 다음 메시지로 시장 폐장 제어 확인:

| 로그 레벨 | 메시지 예시 | 의미 |
|-----------|------------|------|
| `INFO` | `모든 시장 폐장 (KRX·US 모두 폐장) — 5분 대기 후 재확인` | 전체 대기 진입 |
| `DEBUG` | `시장 상태: KRX 개장 · US 폐장` | 일부 개장 (정상 순회) |
| `DEBUG` | `시장 폐장 — 건너뜀: QQQM (US)` | 개별 종목 건너뜀 |

### 7-4. 국내 + 해외 혼합 운용 시 권장 방법

국내 ETF(예: KODEX200 `069500`)와 해외 ETF(예: `QQQM`)를 **동시에 활성화** 해도 됩니다.
시스템이 시간대별로 자동 분리하여:

- **KRX 개장 시간 (09:00-15:30 KST)**: 국내 종목만 처리, 해외 종목 자동 건너뜀
- **US 개장 시간 (22:00-07:00 KST)**: 해외 종목만 처리, 국내 종목 자동 건너뜀
- **양쪽 폐장 시간**: 5분 단위 대기, API 호출 없음

### 8-2. 하락장 적합 전략 조합

#### 🛡️ 방어적 운용 — 추천 전략

| 전략 | 하락장 적합성 | 설정 조정 포인트 |
|------|-------------|----------------|
| **추세 필터** (`trend_filter`) | ✅ 최고 | 장기 MA 200, 단기 5. 200일선 위에서만 매수 → 하락장 자동 차단 |

| 전략 | 이유 |
|------|------|
| **변동성 확장** (`volatility_expansion`) | 하락 급락 시 변동성 확장 → 매수 신호 오발 위험 |
| **이격도** (`deviation`) | MA보다 하락하면 매수 신호 → 추가 하락장에서 손실 누적 |
| **강한 종가** (`strong_close`) | 반등 시도 후 재하락 패턴에서 오시그널 |
| **52주 신고가** (`fifty_two_week_high`) | 하락장에서 돌파 시도 = 덫 가능성 |
| **모멘텀** (`momentum`) | 하락 모멘텀을 상승으로 오해 가능 |

## 9. 자주 묻는 질문 (FAQ)

### Q. KODEX 200과 ACE 200 같은 동일 성격 ETF를 같은 전략 카드에 등록해도 되나요?

**됩니다.** 모든 전략(1~11번 포함)은 종목별로 가격 이력과 포지션 플래그를 독립적으로 관리합니다.

각 전략의 내부 상태는 `종목코드 → 상태` 형태의 HashMap으로 관리되므로,  
KODEX 200과 ACE 200의 가격 이력이 서로 섞이거나 포지션 플래그가 충돌하지 않습니다.

```
[이격도 전략] 대상 종목: [KODEX200 069500] [ACE200 360750]
  → 069500의 MA 이력과 360750의 MA 이력이 완전히 분리됨
  → 한 종목의 매수/매도 신호가 다른 종목에 영향을 주지 않음
```

종목별로 주문 수량만 다르게 설정하고 싶다면 **가격 조건 매매(전략 12)** 를 사용하세요.  
→ [5-9. 다중 종목 등록 — 종목별 독립 상태 지원](#5-9-다중-종목-등록--종목별-독립-상태-지원) 참고

### Q. 체결 내역에서 수수료는 어떻게 계산되나요?

KIS Open API의 체결 이벤트(`H0STCNI0` WebSocket) 및 체결 내역 조회(`TTTC8001R`)는 **건별 수수료를 응답하지 않습니다.** (output2의 `prsm_tlex_smtl`은 조회 기간 전체 합산만 제공)

따라서 AutoConditionTrade는 체결 시 **표준 수수료율 기반 추정값**을 자동 계산하여 기록합니다.

#### 추정 수수료 계산 방식

| 구성 | 요율 | 적용 |
|------|------|------|
| 위탁수수료 | 0.015% | 매수·매도 모두 |
| 증권거래세 | 0.20% | 매도 시에만 |

- **매수**: 체결금액 × 0.015%
- **매도**: 체결금액 × (0.015% + 0.20%) = 체결금액 × 0.215%

#### 표시 위치

| 화면 | 위치 |
|------|------|
| Dashboard | "당일 순손익 (수수료 차감)" 카드 하단 "수수료 XXX원" 표시 |
| History 탭 | 체결 내역 테이블 "수수료" 컬럼 |
| Discord 알림 | 체결 메시지에 "(수수료: XXX원)" 포함 |

#### 주의사항

- 이 값은 **추정치**이며, 실제 수수료는 계좌 유형(우대/일반)·종목별 특수조건에 따라 다를 수 있습니다.
- **실제 확정 수수료**는 KIS 증권 앱 → 거래내역 → 수수료 항목에서 확인하세요.
- `net_profit(순손익) = gross_profit + gross_loss - fees_paid` 공식으로 수수료가 당일 손익에 자동 반영됩니다.
